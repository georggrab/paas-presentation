### Which technologies exist - heroku - openshift - Amazon Web Services (AWS) - IBM Bluemix - Microsoft Azure - Salesforce App Cloud
### Heroku (the one we demo'ed) - by Salesforce - Extremely easy to set up; free tier - Horizontal and Vertical scalability - Managed databases - Popular Programming Frameworks supported out of the box. - Software available as add-on dynos
### Heroku pricing examples #### Dyno (Smallest deployable Unit) - $25 for 512MB Ram, 1 vCore - $200 for 2.5G Ram, 4 vCore
### Openshift * by Redhat * very new (just out of Beta) * free tier * Managed Kubernetes * Deployment of Kubernetes Pods * Cluster extendable by addons - Chaos Monkeys! - Logging, Ingress, Monitoring, ...
### Openshift pricing #### Very reasonable free tier: * 1G Ram, 2 vCPU, burstable #### Premium tier starting at $50 * 2G Ram, 4 vCPU * $25/GiB Ram / month * $1/Gib Persistant Storage / month
### Amazon AWS Elastic Beanstalk * Similar to Heroku in terms of supported Programming Languages / Frameworks. * Beanstalk itself is completely free * Pricing is translated to cost of underlying EC2 Instances / S3 Storage, ELB, ...
### Azure * AKS, which is managed Kubernetes * App Service, which is similar to Heroku
### When to use PaaS? | pro's | con's | | ------------- |:-------------:| | easy to deploy application | less flexibility than IAAS, more flex than CAAS | | no infrastructure work involved | more expensive as IAAS | | fast scale-up with service discovery | gets more expensive as app grows | | CI/CD often integrated | gets more expensive as app grows| | easily extendable (monitoring, logging, databases....) | vendor lock in / opinionated / architecture dependance |
