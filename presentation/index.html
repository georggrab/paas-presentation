What is a PaaS?

  • cloud computing service
  • provides runtime environment
  • often paired with additional services

Why should I use PaaS?

PaaS types and services

Types:

  • public
  • private
  • hybrid

Services:

  • Communications Platform as a Service
  • Mobile Platform as a Service
  • Open PaaS

PaaS - Application Requirements

Apps should be:

  • stateless
  • designed for ci deployment
  • scalable
  • fault tolerant
  • of micorservice pattern
### Which technologies exist - heroku - openshift - Amazon Web Services (AWS) - IBM Bluemix - Microsoft Azure - Salesforce App Cloud
### When to use PaaS? - Pros | pro's | con's | | ------------- |:-------------:| | easy to deploy application | less flexibility than IAAS, more flexibility than SAAS | | no infrastructure work involved | more expensive than IAAS | | fast scale-up with service discovery | gets more expensive as app grows | | CI/CD often integrated | gets more expensive as app grows| | easily extendable (monitoring, logging, databases....) | vendor lock in / opinionated / architecture dependance |
